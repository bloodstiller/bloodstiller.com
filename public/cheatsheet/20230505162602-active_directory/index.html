Active Directory Overview:

  • :
  • :
  • :
  • :

_

  • Purpose: Directory service for Windows network environments.
  • Structure: Distributed, hierarchical for centralized management.
    • Resources Managed: Users, computers, groups, network devices, file shares, group policies, devices, trusts.
  • Functions: Provides authentication and authorization in a Windows domain environment.

Security Concerns:

  • Attacks: Increasingly targeted in recent years.
  • Design Flaws: Backward-compatible, not “secure by default,” prone to misconfiguration.
  • Weakness Exploitation: Can be leveraged to move laterally/vertically, gain unauthorized access.
  • Accessibility: Large read-only database, accessible to all users.
  • Enumeration: Basic user accounts can enumerate most objects, making security critical.
  • Attack Examples: Standard domain user accounts can perform attacks; defense-in-depth strategy required.

If Windows Server 2008, Windows 7 or 8 is found:

  • This indiciates that there are legacy systems still running.
  • These systems are suscetible to classic attacks such as Eternal Blue which will provide a system level shell.

Active Directory checklist [0/0]:

  • Checklists are the bare minimum which should be done on a target, not the be all & end all.

    • Identifying Hosts [0/3]:

      1. Run or passively to listen for traffic:

        • sudo responder -I <interface> -A
          • This uses passive analysis mode & does not send any poisoned packets.

      2. Run on subnet to find targets:

        • fping -asgq <ip/range> > hosts.txt
          • This will display to the screen but also output just hosts to a text file that can be ingested into nmap.

      3. Run initial scan on hosts file output from fping:

        • command: sudo nmap -v -A -iL hosts.txt -oA ~/<output>/<location>/<filename(noextensionneeded)>
        • command: sudo proxychains nmap -v -A -sT -iL hosts.txt -oA ~/<output>/<location>/<filename(noextensionneeded)>

    • Identifying Users [0/2]:

      • Enumerate users with
        • Command: kerbrute userenum -d <domain>.LOCAL –dc <dcIP> /word/list -o <outputFile>
        • Example kerbrute userenum -d INLANEFREIGHT.LOCAL –dc 172.16.5.5 jsmith.txt -o valid_ad_users
        • NOTE: When enumerating AD users the jsmith.txt & jsmith2.txt from the collection is very good.
