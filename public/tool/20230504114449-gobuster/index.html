Used For: Directory Busting

GoBuster is a versatile tool that allows for performing DNS, vhost, and directory brute-forcing. The tool has additional functionality, such as enumeration of public AWS S3 buckets. For this module’s purposes, we are interested in the directory (and file) brute-forcing modes specified with the switch dir. Let us run a simple scan using the dirb common.txt wordlist.

Command Cheat Sheet:

  • Dirbusting:

    • Command: gobuster dir -u <http://> -w <wordlist>
    • Example: gobuster dir -u <http://> -w /usr/share/wordlist/dirb/common.txt

  • DNS Busting:

    • Command: gobuster dns -d <url> -t 50 -w <wordlist>
    • Example: gobuster dns -d googl.com -t 50 -w /usr/share/SecLists/Discovery/DNS/namelist.txt

Create custom busting word list using pattern Matching with Gobuster :

  • Patterns option is particularly useful for discovering subdomains following known naming conventions.
  • Wordlist from Seclists repository can be used with gobuster for pattern-based word searching.
  • Example pattern:
    • lert-api-shv-{NUMBER}-sin6.facebook.com.
    • lert-api-shv-819819-sin6.facebook.com.
  • Create a patterns.txt file with the discovered patterns.

__

  1. Make a patterns.txt with the patterns found from enumeration with tools such as
  2. See below:
export TARGET="domain.com"
export NS="discovered DNS server"
export WORDLIST="/usr/share/seclists/Fuzzing/Amounts/<desired_wordlist.txt>"
touch gobuster_output.txt
gobuster dns -q -r "${NS}" -d "${TARGET}" -w "${WORDLIST}" -p ./patterns.txt -o "gobuster_${TARGET}.txt"
  • Command Breakdown:
    • dns: Launch the DNS module
    • -q: Don’t print the banner and other noise.
    • -r: Use custom DNS server
    • -d: A target domain name
    • -p: Path to the patterns file
    • -w: Path to the wordlist
    • -o: Output file
