Overview:

scp plaintext@192.168.49.128:/root/myroot.txt .

  • scp: scp stands for “secure copy” and is a command-line tool used to securely transfer files between local and remote systems.

  • plaintext@192.168.49.128: This specifies the username and IP address of the remote system you want to connect to. In this case, the username is plaintext, and the IP address is 192.168.49.128. You may need to replace plaintext with the appropriate username.

  • :/root/myroot.txt: This is the path of the file on the remote system that you want to copy. In this case, it’s /root/myroot.txt. This assumes that the file myroot.txt is located in the /root directory on the remote system. Adjust the path according to your specific file location.

  • .: This denotes the current directory on the local system where the file will be copied. In this case, it represents the current directory. The dot (.) is commonly used to indicate the current working directory.

So, when you execute this command, it establishes a secure connection to the remote system at 192.168.49.128 using the username plaintext. It then copies the file myroot.txt from the remote system’s /root directory to the current directory (.) on your local system .

Uploading Files Using SCP:

  • Copy Local File to Remote Server:

    • Command: scp /path/to/local/file username@remote_host:/path/to/remote/directory

  • Copy Local File to Remote Server on Specific Port:

    • Command: scp -P port_number /path/to/local/file username@remote_host:/path/to/remote/directory

  • Copy Directory Recursively:

    • Command: scp -r /path/to/local/directory username@remote_host:/path/to/remote/directory

  • Using SSH Keys:

    • Command: scp -i id_rsa /home/kali/Desktop/LinuxTools/agent root@10.10.110.100:/tmp/my_agent

Specifying different port:
